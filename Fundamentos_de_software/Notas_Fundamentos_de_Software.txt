Las conexiones cableadas en internet son muy caras, y el tráfico de datos es compartido entre todos, siendo de esta manera caro conectarse a los internet exchange points (red cableada mundialmente).
La ultima milla es la conexión entre el modem y el isp.
Si hay una petición de datos entre 2 dispositivos con el mismo isp, lo que habitualmente pasa es transferir los datos a través de una MAN (Metropolitan Area Network), sin pasar por el IXP (Internet Exchange Point), de esta manera el ISP no paga por el uso del IXP.
Usando este principio los ISP pueden ofrecer servicios locales de alta velocidad de transferencia.
Cuando es necesario conectarse por IXP, los servidores crean prioridad, angostando la conexión a través de routers ,firewalls y network shapers, para asignar la prioridad a emails, llamadas por internet o acciones empresariales (de donde mas fondos obtienen) y bajan la prioridad a videos o descargas de archivos secundarios como los de bitorrent.
Quality of Service (también llamado Network Shaping o Throttling) sirve para regular que sitios tienen preferencia a una mayor velocidad y cuales no, esto se hace en la ultima milla.
Los dueños de los dominios en un intento para contrarrestar el efecto del Throttling, implementaron un concepto llamado CDN ( Content Delivery Networks), con lo cual basicamente hacen una copia de los archivos que son estáticos en distintas partes del mundo para asi tener mas cerca de cada usuario la información y esta pueda viajar sin problemas.
La conexión de los servidores de internet se mide en Megabits Mb, pero la información viene en bytes, por ende son velocidades mas lentas de lo que esperaríamos en tiempo de descarga.
Con una conexión de 10Mb/s solo se pueden transmitir 1.25 MB/s.
Además la velocidad de red se ve reflejada por cuanto tiempo tarda la conexión en establecerse y pues fisícamente tiene que viajar de un lugar a otro y para ello se usa el ping que es el tiempo que tarda la conexión en establecerse.

Esto se divide en Backend y Frontend.
El lenguaje mas popular de backend es php, sin embargo existen otros como python, node js, java.
Los servidores mas populares son programas que procesan el código del backend, como nginix, apache, internet information service y node JS.
Un stack es un grupo de tecnologías, uno de estos es LAMP.
El ciclo de un stack es asi:
El navegador le pasa la url al servidor DNS (DIN DNS),el servidor DNS cambia el domino de platzi por una IP,luego el servidor de platzi recibe la url con apache o nginix, luego los datos son pasados al backend, en este caso go, python y node js, estos procesan la información para hacer una petición a la base de datos, obtienen los datos y mandan de respuesta una página con html, css y js, el cual le mandan al navegador.
El código del frontend siempre es visible, por lo cual este le hace peticiones al backend el cual se encargar de obtener la información y a la vez protegerla.
Existen muchos metódos para conectarse desde el el frontend (html, css y Javascript) como el get, post, ajax, sockets.
Get manda los datos en la url, Post los manda a través de los headers http.
Para hacer cambios sin recargar la página ya sea por usar post o get se puede usar AJAX, el cual en el mismo javascript hace una petición asíncrona y modifica una cierta parte del html o css.
Un stack popular es MEAN el cual está formado por mongoDB, express, AngularJS (libreria para frontend) y Node-JS.
